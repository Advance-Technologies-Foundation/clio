COMMAND TYPE
    Development commands

NAME
    update-page - Update the raw schema body of a Freedom UI page

ALIASES
    page-update

DESCRIPTION
    The update-page command validates and saves the raw JavaScript body of a
    Freedom UI page schema. Pass the full body string directly, typically
    after reading raw.body from get-page.

    After a successful non-dry-run save, update-page also attempts a
    best-effort live Designer Presence notification so active Creatio
    designers can be warned that the page was saved outside their session.
    This live push requires forms-auth browser-session cookies (login/password
    backed flow). In OAuth-only or credential-less environments the page save
    still succeeds, and the structured response includes a warning when the
    live notification is skipped or fails.

    When the body contains #ResourceString(key)# macros, update-page can
    register missing child-schema localizableStrings before saving. Pass
    --resources when you need explicit captions, or let clio derive captions
    automatically for missing Usr* keys.

    Keep each field control bound to the declared view-model attribute from
    viewModelConfig / viewModelConfigDiff. If validator or handler logic
    is moved to a different attribute for the same field, rebind the
    control to that attribute too.
    If the control is inherited from a parent schema and there is no local
    entry for it in viewConfigDiff, add a local merge for that control
    name instead of editing a non-existent local insert.

    VALIDATION RULES:
    - SCHEMA_CONVERTERS keys (object form) must follow VendorPrefix.Name
      format (e.g., usr.MyConverter).
    - SCHEMA_HANDLERS must be an array of {request, handler} entries; each
      request value must follow VendorPrefix.Name format (e.g.,
      crt.HandleViewModelInitRequest, usr.HandleSomeRequest).
    - SCHEMA_VALIDATORS keys (object form) must follow VendorPrefix.Name
      format (e.g., usr.RequiredValidator).
    - User-visible text (label, caption, title, tooltip, placeholder) in
      viewConfigDiff must be a localizable-string binding, not an inline
      literal. Bind via $Resources.Strings.<Key> (or #ResourceString(<Key>)#
      for grid column captions / validator messages) and register the key's
      default-language value with --resources. Inline literals are rejected.
    A malformed VendorPrefix.Name causes a Creatio runtime error:
      "Error when register X. Type property should have format VendorPrefix.TypeName"
    - Call `clio get-guidance --name page-schema-converters` for converter details
    - Call `clio get-guidance --name page-schema-handlers` for handler details
    - Call `clio get-guidance --name page-schema-validators` for validator details
    - Call `clio get-guidance --name page-schema-resources` for localization details

    CONFLICT DETECTION:
    update-page guards against overwriting schema changes made outside your
    session (for example, edited in the Creatio designer). It obtains the
    baseline checksum in one of two ways:
      - automatically, from the .clio-pages/{schema-name}/meta.json baseline
        written by a previous get-page for the same environment. No extra flags
        are needed — this is what protects AI-agent CLI flows that read a page
        with get-page and then save it with update-page.
      - explicitly, from --expected-checksum (takes precedence over the on-disk
        baseline).
    Before saving, the baseline is compared against the current
    SysSchema.Checksum of the editable schema. On a mismatch the save is blocked
    with a structured conflict response (conflict: true + conflictDetails)
    instead of silently overwriting the external changes. Re-run get-page,
    re-apply your edit on top of the fresh body, then retry — or pass --force to
    deliberately overwrite. After a successful save the on-disk baseline is
    refreshed automatically, so consecutive updates in the same session do not
    false-conflict. A small race window between the check and the save remains
    (last write wins).

    WRITE MODES:
    --mode replace (default) saves the body verbatim. --mode append loads the
    current schema body from the server and merges your incoming fragment into
    it (viewConfigDiff dedupes by name, handlers by request). Append requires the
    diff form: a full-config body (the SCHEMA_VIEW_MODEL_CONFIG / SCHEMA_MODEL_CONFIG
    markers, instead of the *_DIFF markers) cannot be merged and is rejected with
    an actionable hint — use --mode replace for such a body.

SYNOPSIS
    clio update-page [options]

OPTIONS
    --schema-name                      Page schema name to update

    --body                             Full raw JavaScript schema body (a
                                       string with /**MARKER*/ pairs, not a
                                       structured object)

    --body-file                        Path to a file containing the body.
                                       Alternative to --body for large bodies

    --mode                             Write mode: 'replace' (default) or
                                       'append' (merge with the current body).
                                       Append requires the diff form; a full-
                                       config body is rejected — use 'replace'

    --dry-run                          Validate only and do not save

    --resources                        Valid JSON object of resource key-value
                                       pairs for #ResourceString(key)# macros
                                       Malformed JSON is rejected during
                                       validation

    --optional-properties              JSON array of {key, value} objects to
                                       merge into schema optionalProperties,
                                       e.g. '[{"key":"entitySchemaName",
                                       "value":"UsrMyEntity"}]'

    --expected-checksum                Baseline SysSchema checksum of the
                                       editable schema (from get-page). Blocks
                                       the save with a conflict error when the
                                       server-side checksum differs

    --force                            Skip the external-modification check
                                       and deliberately overwrite out-of-band
                                       changes

    --uri                    -u       Application uri

    --Password               -p       User password

    --Login                  -l       User login (administrator permission required)

    --Environment            -e       Environment name

    --Maintainer             -m       Maintainer name

EXAMPLE
    clio update-page --schema-name UsrTodo_FormPage --body "<raw body>" --dry-run true -e dev
        validate a raw Freedom UI body without saving it

    clio update-page --schema-name UsrTodo_FormPage --body "<edited raw body>" -e dev
        save the edited raw Freedom UI body to the registered dev environment

    clio update-page --schema-name UsrTodo_FormPage --body "<edited raw body>" --resources "{\"UsrDetailsTab_caption\":\"Details\"}" -e dev
        save the page and register the missing child-schema localizable string

    clio update-page --schema-name UsrTodo_FormPage --body "<edited raw body>" --optional-properties "[{\"key\":\"entitySchemaName\",\"value\":\"UsrTodo\"}]" -e dev
        save the page and merge custom optional properties into the schema
