describe-process (alias: dp)

Reads an existing Creatio business process and returns a STRUCTURED graph
(elements, flows, and process parameters) instead of the raw escaped metadata,
so it can be explained in plain language. This is the inverse of process generation
("read & explain").

USAGE
  clio describe-process --process-code <code> -e <environment>
  clio describe-process --process-uid <guid> -e <environment>
  clio describe-process --process-caption "<caption>" -e <environment>
  clio dp --process-code UsrProcess_493d4c9 -e my-env

OPTIONS
  --process-code      Process code (schema Name), e.g. UsrProcess_493d4c9.
  --process-uid       Process UId (GUID).
  --process-caption   Process caption (display name).
  --culture           Culture for localized captions (default: en-US).
  -e, --environment   Registered clio environment name.

Provide EXACTLY ONE of --process-code / --process-uid / --process-caption.

OUTPUT
  Structured JSON with:
    name, caption, schemaUId
    elements   - [{ id, name, caption, type, userTaskName?, position, parameters }]
                 type = runtime class (e.g. ProcessSchemaUserTask); userTaskName = the
                 specific user-task schema (e.g. ActivityUserTask), incl. custom ones.
    flows      - [{ source, target, kind }]  (kind: sequence | conditional | default)
    parameters - [{ name, uid, source, value }]  (process-level inputs / variables)
  Each element/process parameter carries its value source (None | ConstValue | Mapping |
  Script | SystemValue | ...) and the raw value/expression.

NOTES
  - The read is performed server-side by the ProcessDesignService (clioprocessbuilder)
    package via platform managers (runtime instance, with a design-time fallback for
    file-design-mode). The target environment must have the package installed.
  - Element typing is universal (real object model), so custom user tasks resolve to their
    actual schema name.
  - Parameter value expressions are surfaced verbatim (incl. formula [#...#] references);
    translating them into plain language is planned future work.
