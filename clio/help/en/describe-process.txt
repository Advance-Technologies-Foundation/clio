describe-process (alias: dp)

Reads an existing Creatio business process and returns a STRUCTURED graph
(elements, flows, and process parameters) instead of the raw escaped metadata,
so it can be explained in plain language. This is the inverse of process generation
("read & explain").

USAGE
  clio describe-process --process-code <code> -e <environment>
  clio describe-process --process-uid <guid> -e <environment>
  clio describe-process --process-caption "<caption>" -e <environment>
  clio dp --process-code UsrProcess_493d4c9 -e my-env

OPTIONS
  --process-code      Process code (schema Name), e.g. UsrProcess_493d4c9.
  --process-uid       Process UId (GUID).
  --process-caption   Process caption (display name).
  --culture           Culture for localized captions (default: en-US).
  -e, --environment   Registered clio environment name.

Provide EXACTLY ONE of --process-code / --process-uid / --process-caption.

OUTPUT
  Structured JSON with three sections:
    elements   - [{ id, dataId, type, label, parameters }]
    flows      - [{ source, target, kind }]  (kind: sequence | conditional | default)
    parameters - process-level parameters (name, type, direction, caption)

NOTES
  - Reuses the existing ProcessSchemaRequest read path; element types use the same
    vocabulary as the `process-modeling` guidance and `validate-process-graph`.
  - v1 does NOT decode element filter / mapping expressions — it returns structure,
    element types, flows, and basic parameters only.
