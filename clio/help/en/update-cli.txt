COMMAND TYPE
    Application management

NAME
    update-cli - Update clio to the latest available version

SYNOPSIS
    update-cli [OPTIONS]
    update [OPTIONS]

DESCRIPTION
    Checks for a newer version of clio on NuGet.org and updates the installation
    if an update is available. By default, the command operates in interactive mode,
    showing the current and latest versions and prompting the user for confirmation
    before updating.
    
    Recommended to use the latest version of clio for bug fixes and new features.

OPTIONS
    -g, --global            Install clio globally (default: true)

    -y, --no-prompt         Skip confirmation prompt and proceed with update
                            automatically. Useful for automated scripts.

EXAMPLES
    # Interactive update (prompts user)
    update-cli

    # Automatic update without confirmation
    update-cli --no-prompt

    # Using short alias
    update -y

BEHAVIOR
    - Checks current installed version and latest version on NuGet.org
    - If already on latest version, displays confirmation message and exits
    - If update available and not --no-prompt, displays versions and prompts for confirmation
    - On user confirmation or with --no-prompt, runs: dotnet tool update clio -g
    - Verifies the new version is installed correctly
    - Reports success or failure with appropriate exit codes

EXIT CODES
    0   Successful update or already on latest version
    1   User cancelled or update failed
    2   Error checking for updates (network issue, etc.)
