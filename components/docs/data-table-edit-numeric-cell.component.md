# How to Add a DataTable Edit Numeric Cell (`crt.DataTableEditNumericCell`) to a DataTable Column

> Audience: code agent configuring an editable numeric column in a `crt.DataTable` in a Creatio Freedom UI
> page schema.
>
> `crt.DataTableEditNumericCell` is the inline numeric editor shown when a user activates an integer, float,
> or money-type cell in an editable `crt.DataTable` column. It is **not** inserted via `viewConfigDiff` — it
> is set as the `editingCellView` property on the column definition.

## Metadata

- **Category**: display
- **Container**: no
- **Parent types**: column `editingCellView` slot inside `crt.DataTable`
- **Typical children**: none

---

## 1. Mental model — the 1 place you must edit

| # | Section | What you add |
|---|---|---|
| 1 | `viewConfigDiff` — `crt.DataTable` column's `editingCellView` | An object `{ "type": "crt.DataTableEditNumericCell", "control": "$Attr" }`. **Always present.** |

There is **no** standalone `insert` op for this cell. No `modelConfigDiff` or `viewModelConfigDiff` changes are
needed for the cell itself.

### 1.1 Naming convention

```
// no standalone name — inline config
// bound attribute follows DataTable column binding convention, e.g.:
$DetailDS_Amount
```

---

## 2. Step-by-step recipe

### 2.1 Add `editingCellView` to the DataTable column

```jsonc
{
  "operation": "insert",
  "name": "MyDataTable",
  "parentName": "MainContainer",
  "propertyName": "items",
  "index": 0,
  "values": {
    "type": "crt.DataTable",
    "items": "$MyDetailDS",
    "columns": [
      {
        "id": "col-abc123",
        "code": "Amount",
        "caption": "#ResourceString(MyDataTable_column_Amount_caption)#",
        "dataValueType": 6,
        "cellView": {
          "type": "crt.TableTextCell",
          "value": "$MyDetailDS_Amount"
        },
        "editingCellView": {
          "type": "crt.DataTableEditNumericCell",
          "control": "$MyDetailDS_Amount",
          "format": { "decimalPrecision": 2 }
        }
      }
    ]
  }
}
```

## 5. Copy-paste minimal example

No direct PackageStore match was found; numeric cells are typically generated by `crt.EditTypedValueCell`
dynamically. Use the column pattern below based on the standard `editingCellView` convention:

```jsonc
// Integer column
{
  "id": "col-qty",
  "code": "Quantity",
  "caption": "Quantity",
  "dataValueType": 5,
  "cellView": {
    "type": "crt.TableTextCell",
    "value": "$MyDetailDS_Quantity"
  },
  "editingCellView": {
    "type": "crt.DataTableEditNumericCell",
    "control": "$MyDetailDS_Quantity"
  }
}
```

```jsonc
// Money column with 2 decimal places
{
  "id": "col-amount",
  "code": "Amount",
  "caption": "Amount",
  "dataValueType": 6,
  "cellView": {
    "type": "crt.TableTextCell",
    "value": "$MyDetailDS_Amount"
  },
  "editingCellView": {
    "type": "crt.DataTableEditNumericCell",
    "control": "$MyDetailDS_Amount",
    "format": { "decimalPrecision": 2 }
  }
}
```

## 6. Driving from page state

The `readonly` input is `propertyBindable`:

```jsonc
"editingCellView": {
  "type": "crt.DataTableEditNumericCell",
  "control": "$MyDetailDS_Amount",
  "readonly": "$MyTable_readonly"
}
```

## 7. Common pitfalls

1. **Using an `insert` op instead of `editingCellView`** — this cell is not a standalone view element.
2. **Binding `control` to a raw number** — `control` must receive a `FormControl`; bind via `$AttributeName`.
3. **Omitting `format` for money/float columns** — without `decimalPrecision` the input allows unlimited
   decimal digits which may not match the column's schema precision.
4. **Using for text/boolean columns** — use `crt.DataTableEditTextCell` for text and `crt.TableBooleanCell`
   for booleans; `crt.EditTypedValueCell` can dispatch to the correct cell type automatically.
5. **Forgetting `cellView`** — `cellView` and `editingCellView` are independent; without `cellView` the
   display-mode cell renders blank.

---

## 8. Quick checklist

- [ ] `editingCellView` is set inside the DataTable column, not as a standalone `insert` op.
- [ ] `control` is bound to a `$AttributeName` referencing a numeric `FormControl`.
- [ ] `cellView` is also set on the same column for the read-only display state.
- [ ] `format.decimalPrecision` is set for float/money columns.
- [ ] `readonly` is bound or set to a literal value as needed.
