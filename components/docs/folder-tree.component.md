# How to Add a Folder Tree (`crt.FolderTree`) to a Freedom UI Page

> Audience: code agent inserting a `crt.FolderTree` into a Creatio Freedom UI page schema.
>
> A `crt.FolderTree` is a hierarchical folder navigation panel. It renders a virtual-scroll
> tree of folders and emits events for every folder CRUD operation (add, rename, copy, move,
> delete, change rights, convert to static). Adding one only requires a single `viewConfigDiff`
> insert — the `_filterOptions` block is auto-generated by the create command.

## Metadata

- **Category**: filter
- **Container**: no
- **Parent types**: `crt.FlexContainer`, `crt.GridContainer`
- **Typical children**: none

---

## 1. Mental model — the 1 place you must edit

| # | Section | What you add |
|---|---|---|
| 1 | `viewConfigDiff` | A single `insert` op with `type: "crt.FolderTree"`, `sourceSchemaName`, `rootSchemaName`, and `_filterOptions`. **Always present.** |

`crt.FolderTree` is **view-only** — no `modelConfigDiff` or `viewModelConfigDiff` required.
The `_filterOptions` block is identical to `crt.FilterBuilderSource` / `crt.FilterBuilderToggler`
and is added automatically by `crt.AddFilterViewItemCommand`.

### 1.1 Naming convention

```
FolderTree_<id>              // view element name; <id> is any short unique slug
FolderTree_<id>_Value        // internal filter binding name (used in _filterOptions.from)
FolderTree_<id>_Filter       // exposed attribute name (used in _filterOptions.expose[].attribute)
```

---

## 2. Step-by-step recipe

### 2.1 Insert the view element (`viewConfigDiff` entry)

```jsonc
{
  "operation": "insert",
  "name": "FolderTree_abc123",
  "parentName": "MainContainer",
  "propertyName": "items",
  "index": 0,
  "values": {
    "type": "crt.FolderTree",
    "sourceSchemaName": "FolderTree",
    "rootSchemaName": "Contact",
    "_filterOptions": {
      "expose": [
        {
          "attribute": "FolderTree_abc123_Filter",
          "converters": [{ "converter": "crt.FolderFilterAttributeConverter" }]
        }
      ],
      "from": "FolderTree_abc123_Value"
    },
    "layoutConfig": { "width": 328 },
    "visible": true
  }
}
```

## 3. Property reference

Full `inputs`, `outputs`, `default`, `values`, `deprecated`, and `designerDefaults` for
`crt.FolderTree` are in `ComponentRegistry.json` under `componentType: "crt.FolderTree"`. This guide
covers only the assembly mechanics.

## 4. Shape of types not in `references.typeDefinitions`

`items` and `favoriteItems` accept `DataItem[]` or a `BaseViewModelCollection`. The component
reads `Id`, `Name`, `ParentId`, `FilterData`, `FolderTypeId` (and `FolderTreeId` for favorites)
from each item asynchronously.

`activeFolderChanged` emits `ActiveFolderChangedEvent`:

```ts
interface ActiveFolderChangedEvent {
  id: string;             // empty string when root is selected
  filterData: string;     // JSON string of the folder filter; '{}' when root
  folderTreeId: string;
  folderTypeId?: string;
  rootSchemaName: string;
  name: string;
  isFavorite: boolean;
}
```

## 6. Driving from page state

`readonly` is `propertyBindable` — bind to a `$Attribute` to toggle the tree between read-only
and editable mode at runtime:

```jsonc
"readonly": "$FolderTree_readonly"
```

`activeFolderId` and `expandedItems` should be persisted in page attributes and passed back to
the component to maintain selection/expansion state across navigations.

---

## 7. Common pitfalls

1. **`sourceSchemaName` not set** — the component cannot check add/edit rights; defaults to `"FolderTree"` per `defaultPropertyValues`; set explicitly for non-default folder schemas.
2. **`rootSchemaName` missing** — folder filter dialogs and move-folder dialogs require it; always set to the page entity schema name.
3. **`_filterOptions.from` not matching `<name>_Value`** — the active-folder filter will not propagate to datasource bindings.
4. **`items` / `favoriteItems` not provided** — the tree renders empty; supply these from a datasource response.
5. **`activeFolderId` not persisted** — on page re-render the tree will deselect the active folder; store in a page attribute and re-bind.
6. **`readonly: true` without updating `activeFolderChanged` handling** — in readonly mode the tree still fires `activeFolderChanged`; add/rename/delete events are suppressed.
7. **`useStaticFolders: true` without static-folder support in the data layer** — the "Add static folder" menu item appears but the server-side schema must support static folders.

---

## 8. Quick checklist

- [ ] `insert` op in `viewConfigDiff` with `type: "crt.FolderTree"`, unique `name`, valid `parentName`, `propertyName: "items"`, valid `index`.
- [ ] `sourceSchemaName` set (default `"FolderTree"`).
- [ ] `rootSchemaName` set to the page entity schema name.
- [ ] `_filterOptions.from` is `<name>_Value`; `expose[].attribute` matches the attribute name used in datasource bindings.
- [ ] `layoutConfig.width: 328` (design default).
- [ ] `activeFolderChanged` wired so the parent list can filter by the selected folder.
- [ ] `items` / `favoriteItems` populated from a datasource.
