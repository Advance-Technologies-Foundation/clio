## Key technical learnings (do not re-derive)
- A process MUST have LaneSet→Lane; nodes contained in lane; sequence flows have NO container. Missing lane → "Collection item {schemaUId} not found".
- Perform task (`ActivityUserTask`) needs process `SerializeToDB=true`. Designer sets it for ALL new BPs anyway.
- Element typing: read `element.GetType().Name`; user task → `ProcessSchemaUserTask.SchemaUId` resolved via `ProcessUserTaskSchemaManager.FindItemByUId(uid).Name`.
- Managers: get via `userConnection.ProcessSchemaManager` / `ProcessUserTaskSchemaManager` / `DataValueTypeManager`; or `GetSchemaManager(name)` / `SchemaManagerProvider`. No single "SchemaManager" — abstract `SchemaManager<T>` base + per-type managers; route by `SysSchema.ManagerName`.
- Manager read paths: `FindInstanceBy*` = runtime (DB/compiled); `ForceGetInstance` = needs compiled class; `GetDesignInstance` = needs a design session item (create with public `DesignSchema(uc,uid)` first); `*FromMetaData` methods are `internal` (not callable from package).
- FSD/file-design-mode: BuildProcess saves to FS only (designer sees it), NOT DB → not in `VwProcessLib`, not runtime-runnable until FS→DB load. On non-FSD envs SaveSchema → DB → immediately runnable. This is a real persistence consideration for the feature.
- Palette curation lives ONLY in `SysProcessUserTask` (cols: SysUserTaskSchemaUId, Position) — no per-schema flag. Client manager adds `createExistsFilter("[SysProcessUserTask:SysUserTaskSchemaUId:UId].Id")`.
- No C# `ProcessSchemaElementsInitializer` (client-only); build the lane manually server-side.
- Build/save: `manager.CreateSchema(name, manager.DefSchema, uc, Guid.NewGuid(), false)` → build on `item.Instance` → `manager.SaveSchema(item, uc, true, true)`. Rollback on failure: `manager.RemoveItemByUId(item.UId, uc)`.
