# Демо-скрипт (Блок 1): «Агент будує застосунок Invoices»

Готова до репетиції шпаргалка для живої демки на SC «Clio MCP: capabilities».
Ціль — за ~10 хв показати композицію всіх 6 областей Блоку 1 (додаток → об'єкти → сторінки →
хендлери/валідатори → компоненти → бізнес-правила) і **посіяти** ньюанси Блоку 2.

> ⚠️ Точний вибір інструментів і аргументів — рішення агента. Нижче — **очікувані** tool-calls, якщо
> агент іде за guidance. **Обов'язково прорепетируйте проти реального середовища** й запишіть повний
> прогін як fallback: живі демо падають, запис — ні.

---

## Перед виступом (pre-flight)

- [ ] Середовище `dev` зареєстроване й здорове: `clio get-info -e dev` проходить; cliogate встановлено.
- [ ] MCP-клієнт (Claude Desktop / Claude Code / IDE) підключений до `clio mcp-server` (stdio), середовище `dev` — активне.
- [ ] Інструменти запису **allow-listed** у клієнті (щоб не ловити confirmation-промпти в ефірі):
      `create-app`, `create-entity-schema`, `create-page`, `update-page`, `create-page-business-rules`,
      `compile-creatio`, а також `clio-run` (executor для long-tail).
- [ ] Розкладка екрана: ліворуч — чат агента, **видно лог tool-calls**; праворуч — вкладка Creatio.
- [ ] Шрифт термінала/чату великий (18pt+).
- [ ] **Fallback:** записаний скрінкаст повного прогону + скріншоти кожного кроку.
- [ ] **Reset між прогонами:** видалити застосунок або щоразу новий префікс пакета (`UsrInvoicesDemo2`, …).
- [ ] Прибрати з екрана токени/паролі/чутливі env-name (redaction прикриває помилки, але не сам чат).

**Промпти нижче — українською** (щоб зал читав), але агент однаково розуміє англійську. Значення в
`моноширинному` — це те, що варто називати точно.

---

## Наскрізний меседж (казати між кроками)
Двічі-тричі вголос підкреслити «seed» для Блоку 2:
> «Помітили: **перед** кожною дією агент читає *guidance* і питає *актуальний* каталог компонентів.
> Це не я його прошу — так налаштований сервер. Чому це критично для LLM — у другій половині.»

## Кроки

### Крок 0 — Розігрів / показати зв'язок (~0:30)
- 🎤 **Промпт:** «Покажи мої застосунки Creatio в середовищі `dev`.»
- 🔧 **Очікувано:** `list-apps` (resident tool — працює одразу, без discovery).
- 🗣 **Залу:** «Агент уже підключений до мого dev через MCP. Ось поточні застосунки — зараз додамо свій.»
- 👀 **Показати:** відповідь зі списком; у лозі — один виклик `list-apps`.

### Крок 1 — Створення додатку *(область 1)* (~1:30)
- 🎤 **Промпт:** «Створи застосунок Creatio з назвою `Invoices` і пакетом `UsrInvoices` у середовищі `dev`.»
- 🔧 **Очікувано:** `get-guidance` (`core-rules`, `routing`, далі `app-modeling`) → `create-app`.
- 🗣 **Залу (seed #1):** «Зверніть увагу на лог: **спершу** `get-guidance core-rules` і `routing`, і лише
  потім `create-app`. Агент читає правила гри перед дією.»
- 👀 **Показати:** підтвердження створення; у Creatio (за бажанням) — новий застосунок у списку.

### Крок 2 — Об'єкт: створення *(область 2)* (~2:00)
- 🎤 **Промпт:** «Додай у застосунок Invoices об'єкт `UsrInvoice` з полями: `Number` (текст),
  `Amount` (гроші), `DueDate` (дата).»
- 🔧 **Очікувано:** `get-guidance` (`app-modeling`) → `create-entity-schema`
  (+ можливо `modify-entity-schema-column` / `set-entity-schema-properties`); далі часто `compile-creatio`.
- 🗣 **Залу:** «Агент моделює сутність і колонки. Під капотом — ClioGate, що обходить обмеження
  DataService на схемні об'єкти. Далі — компіляція, це нормальний крок Creatio.»
- 👀 **Показати:** підтвердження; у лозі видно guidance → create-entity-schema → compile.

### Крок 3 — Сторінка: створення *(область 3)* (~2:00)
- 🎤 **Промпт:** «Створи Freedom UI сторінку редагування запису (форму) для об'єкта `UsrInvoice`.»
- 🔧 **Очікувано:** `list-page-templates` → `get-guidance` (`page-creation`, `page-modification`) →
  `create-page` → `get-page`.
- 🗣 **Залу:** «Агент дивиться доступні **шаблони**, читає page-creation guidance і бере правильний
  шаблон форми — не вгадує структуру.»
- 👀 **Показати:** підтвердження + (за бажанням) відкрити сторінку в дизайнері Creatio.

### Крок 4 — Компонент + хендлер *(області 5 і 4)* (~2:00)
- 🎤 **Промпт:** «Додай на форму кнопку `Позначити оплаченим`, яка за кліком ставить статус `Paid`.»
- 🔧 **Очікувано:** `get-component-info` (Button, під версію) → `get-guidance`
  (`page-modification-components`, `page-schema-handlers`) → `update-page` (`viewConfigDiff` + handler).
- 🗣 **Залу (seed #2):** «Дивіться: `get-component-info` повертає компонент **під версію мого
  середовища** — `resolvedFrom=environment`. Агент не вигадує властивостей кнопки. І хендлер він
  пише не навмання, а за guidance page-schema-handlers.»
- 👀 **Показати:** кнопка на формі; у лозі — get-component-info → update-page.
- ➕ *(опц., якщо є час — область 4 повністю)* **Промпт:** «Додай валідатор: `Amount` має бути більше 0.»
  → `get-guidance` (`page-schema-validators`) → `update-page`.

### Крок 5 — Бізнес-правило *(область 6)* (~1:30)
- 🎤 **Промпт:** «Додай на сторінку бізнес-правило: поле `Amount` обов'язкове, коли заповнене `Number`.»
- 🔧 **Очікувано:** `get-guidance` (`business-rules`) → `create-page-business-rules`.
- 🗣 **Залу:** «Декларативне правило, без коду — і зверніть увагу, це саме **page**-business-rule.
  Є повний CRUD окремо для сторінки і для об'єкта.»
- 👀 **Показати:** підтвердження створення правила.

### Крок 6 — Компіляція + показ у Creatio (~1:00)
- 🎤 **Промпт:** «Скомпілюй зміни і дай мені посилання на застосунок.»
- 🔧 **Очікувано:** `compile-creatio` (+ за потреби restart).
- 🗣 **Залу:** «Компілюємо — і відкриваємо в Creatio. Ось наш застосунок: об'єкт, форма, кнопка з
  логікою і бізнес-правило — усе зроблено з чату за ~10 хвилин.»
- 👀 **Показати:** відкрити Creatio, зайти в застосунок Invoices, продемонструвати форму: кнопка
  видима, правило спрацьовує (очистити Number → Amount перестає бути обов'язковим, і навпаки).

## Troubleshooting / ризики
| Симптом | Причина / дія |
|---|---|
| Confirmation-промпт посеред демо | Інструмент не allow-listed → додати в allow-list заздалегідь (див. pre-flight). |
| «GetSchemaDesignItem returned an HTML error page» | Брак залежності пакета → агент має піти в `add-package-dependency` (guidance `package-dependencies`). |
| `get-component-info` віддає `resolvedFrom=latest-fallback` | Не визначилась версія (cliogate/GetSysInfo) → агент попросить підтвердження; для демо краще мати робочий cliogate ≥ `2.0.0.32`. |
| Довга компіляція | Нормально; заповнити паузу наративом про Блок 2 (guidance/версії). |
| Сторінка не відкривається як секція | Форма створена, але не в навігації → опційно `create-app-section`, або відкрити сторінку прямо в дизайнері. |
| Демо «поплило» | Спокійно перемкнутися на записаний fallback і продовжити наратив. |

Головне — показати паттерн «guidance → дія» хоча б раз (seed #1 на Кроці 1).
