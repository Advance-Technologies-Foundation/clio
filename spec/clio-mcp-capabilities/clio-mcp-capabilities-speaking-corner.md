# Speaking Corner: «Clio MCP: capabilities»

Тема й тези для спікінг корнету. Формат — оглядова доповідь для **змішаної аудиторії**
(інженери + продукт/QA/менеджмент), тривалість ~15 хв + Q&A.

---

## TL;DR (одним реченням)

> Clio був CLI для розробника Creatio. Через MCP той самий clio стає «руками і знаннями»
> для AI-агента — і агент може реально будувати на Creatio: від підняття середовища до
> готової сторінки Freedom UI.

---

## Варіанти назви

1. **Clio MCP: як CLI став суперсилою для AI-агентів у Creatio**
2. **Від CLI до AI-агента: можливості Clio MCP**
3. **Clio MCP: міст між AI та low-code Creatio**
4. **Clio MCP: capabilities — що вміє AI, коли в нього є clio**

---

## Абстракт

Clio — це CLI **та** MCP-сервер для платформи Creatio. MCP (Model Context Protocol) — це
стандартний «протокол розетки» між AI та інструментами. Через нього будь-який AI-агент
(Claude, Copilot, десктоп-компаньйон ClioRing) отримує доступ до **понад 120 інструментів**
Clio і може виконувати повний цикл роботи з Creatio: піднімати середовища, деплоїти,
створювати сутності, сторінки Freedom UI, дашборди, бізнес-процеси, теми.

У доповіді — огляд можливостей Clio MCP: **що** він уміє, кілька «розумних» рішень під
капотом (щоб агент не тонув у сотні інструментів і не помилявся), і як це виглядає в дії.

---

## Тези (план доповіді)

### 0. Хук (~1 хв)
- Раніше clio = CLI, яким користується **людина** в терміналі.
- Сьогодні той самий clio через MCP стає інструментом, яким керує **AI**.
- Питання залу: що станеться, коли дати розумному агенту повний, але безпечний доступ до Creatio?

### 1. Що таке Clio MCP (~2 хв)
- **MCP** — відкритий стандарт, як AI-агент «під'єднується» до зовнішніх інструментів
  (єдиний протокол замість купи саморобних інтеграцій).
- **clio** — це і CLI, і MCP-сервер для Creatio одночасно. Одна кодова база — два обличчя.
- Два способи підключення:
  - **Локально (stdio)** — свій агент/IDE на машині розробника.
  - **HTTP-сервіс** — мультитенантний edge для команди/хмари (кілька користувачів, кожен зі своїми
    доступами до своїх середовищ).
- 🔑 Ключова думка: **один і той самий clio** — і для людини в терміналі, і для AI.

### 2. Карта можливостей: що вміє (~4 хв)
Понад 120 інструментів, згрупованих по доменах — фактично покривають повний цикл розробки:

| Домен | Приклади того, що вміє агент |
|---|---|
| **Середовища й деплой** | перевірити інфраструктуру → підняти Creatio + Identity Service |
| **Пакети й воркспейси** | створити workspace, встановити/оновити пакети, керувати залежностями |
| **Дані й схеми** | створити сутність (entity), ESQ-запити, OData CRUD |
| **Freedom UI (сторінки)** | створити/змінити сторінку, додати поля, кнопки, компоненти, бізнес-правила |
| **Дашборди й віджети** | графіки, індикатори, розкладка, права доступу |
| **Бізнес-процеси** | проєктування BPMN-процесів |
| **Брендинг** | створення/зміна кастомних тем |
| **Тестові дані** | DataForge — наповнення середовища даними |
| **Безпека/інтеграції** | OAuth, Identity Service, інтеграційні тести |

- 🔑 Ключова думка: це не окремі трюки, а **повний ланцюжок** — від «нема середовища» до
  «готова робоча сторінка».

### 3. Три речі, які роблять його розумним (~4 хв)
_(без важкої архітектури — на пальцях)_

1. **Не тоне в інструментах.**
   Якщо показати агенту одразу 120+ інструментів, він «захлинається» — губить фокус і робить
   помилки. Тому clio показує лише **~20 найпотрібніших**, а решту віддає **на вимогу**.
   Ефект — у рази менше «шуму» в контексті агента.

2. **Вчить, а не просто дає кнопки.**
   Разом з інструментами clio віддає **понад 50 guidance-статей** і «маршрутизатор задач»:
   він каже агенту не лише *що* натиснути, а *як правильно* зробити конкретну річ у Creatio
   (створити сторінку, додати фільтр, налаштувати права). Знання про платформу — теж частина продукту.

3. **Завжди свіжі компоненти.**
   Каталог компонентів Freedom UI підтягується з CDN **під версію конкретного середовища**.
   Тож агент не вигадує неіснуючі властивості й не пропонує те, чого немає на цій версії платформи.

### 4. Безпечно за замовчуванням (~2 хв)
- **Небезпечні операції позначені** й вимагають підтвердження від людини (видалення, деплой тощо).
- **Секрети не течуть**: паролі, токени й шляхи автоматично вичищаються з помилок і логів.
- **Експериментальні команди — за фіче-тоглами**: увімкнув свідомо → отримав доступ.
- 🔑 Ключова думка: агенту довіряємо рівно стільки, скільки дозволяють запобіжники — і вони вбудовані.

### 5. Видно, що відбувається (~1–2 хв)
- Довгі операції (деплой) **стрімлять живий прогрес**: перелік етапів, статуси, результат.
- Це видно і в чаті агента, і в десктоп-компаньйоні **ClioRing**.
- 🔑 Ключова думка: не «чорна скринька на 10 хвилин», а прозорий процес.

### 6. Демо-ідея (обрати одну) (~2–3 хв)
- **«Створи сторінку»** — попросити агента додати поле або кнопку на сторінку Freedom UI.
  Наочно показує guidance + пошук компонентів у дії.
- **«Підніми середовище»** — deploy lifecycle зі стрімінгом прогресу в реальному часі.
- **«Запитай дані»** — ESQ / OData через природну мову замість ручного запиту.

---

## Головні висновки (для фіналу)

1. **Clio MCP = «руки і знання» для AI на Creatio**, а не тонка обгортка над CLI.
2. **Розумний контекст + guidance + свіжі компоненти** = агент, що будує *правильно*, а не навмання.
3. **Безпечно за замовчуванням + видимий прогрес** = такому агенту можна довіряти реальні задачі.

## Можливі питання від залу (підготуватися)

- **Чим це відрізняється від звичайного виклику CLI командами?**
  → MCP дає агенту структурований опис інструментів, підказки-guidance і безпекові позначки;
  агент сам добирає й комбінує кроки, а не виконує один зашитий сценарій.
- **Хіба безпечно давати AI доступ до продакшн-середовища?**
  → Небезпечні дії потребують підтвердження, секрети вичищаються, а HTTP-режим ізолює доступи
  по користувачах. Це рівень довіри, який контролюємо.
- **Чи прив'язано це лише до Claude?**
  → Ні. MCP — відкритий стандарт, тож підходить будь-який MCP-сумісний клієнт (IDE, Copilot, ClioRing).
- **Що з версіями платформи?**
  → Каталог компонентів визначає версію середовища й віддає відповідний набір; якщо версію
  визначити не вдалося — агента просять уточнити, а не діяти наосліп.

## Нотатки для спікера

- Аудиторія змішана → менше жаргону, більше «навіщо це». Технічні деталі (lazy-schema, транспорти,
  контракти прогресу) тримати як **бекап-слайди** на випадок технічних питань.
- Одна жива демка коштує більше за десять слайдів — якщо є середовище, почати або завершити нею.
- Наскрізна метафора: **«розетка» (MCP) + «руки й знання» (clio)** — тримати її від початку до кінця.
