# business-rules-crud — live testing checklist

Target: local Creatio at `http://localhost:8080` (Supervisor/Supervisor), executed
through the real MCP tools. Mark `[x]` with the covering evidence, or record the failure.

Evidence sources (2026-07-07 run):
- **e2e-E / e2e-P** — `clio.mcp.e2e` sandbox lifecycles
  (`BusinessRules_Should_Complete_Entity_Rule_Crud_Lifecycle_In_Creatio`,
  `…_Page_Rule_Crud_Lifecycle_In_Creatio`, `BusinessRulesRead_Should_RoundTrip_ApplyFilter_Flags_In_Creatio`)
  run live against env `dev` (page lifecycle with `McpE2E:Sandbox:PageSchemaName=UsrBrCrud_FormPage`).
- **CC-A / CC-B** — clean-claude isolated agent runs (Contact filter rules; BrCrud
  set-values/formula/delete-all).
- **raw** — standalone MCP stdio client (batch delete of 13 rules through `clio-run`).
- **unit** — `clio.tests` fixtures (reader/grafter/addon-service/converter/tool mapping).

Defects found and fixed during the run:
1. Server strips the friendly `type: "Const"` marker on read-back → reader now
   dispatches on persisted `typeName` (found by e2e-E).
2. Server strips zero-valued properties → absent `comparisonType` now maps to
   `is-not-filled-in`; grafter matches change triggers with absent `type` (found by e2e-P).
3. Server drops `clearValue`/`populateValue` from the persisted apply-filter action →
   read now derives the flags from the autogenerated child rules (found by CC-A);
   guidance updated so agents verify flags on the action instead of expecting new rules.

> 2026-07-08 contract change: the read envelope (`convertible`/`rule`/`raw`/
> `conversionError`) was flattened - read now returns contract-shaped rules directly and
> fails the whole read on an unrepresentable rule. Evidence notes below that mention
> `convertible` refer to the pre-flattening shape; the covered behaviors are unchanged.

Conventions: E = entity-level, P = page-level. Every update case implies a follow-up
read to verify persisted state (and uId preservation where noted).
