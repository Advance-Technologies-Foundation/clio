# business-rules-crud — implementation plan

Companion to [business-rules-crud-spec.md](./business-rules-crud-spec.md). Paths are
relative to the repo root.

## 1. Contract extension

`clio/Command/BusinessRules/BusinessRuleModels.cs`
- `BusinessRule`: add optional `Name` (`name`), `Enabled` (`enabled`, bool?).
- `BusinessRuleCondition`, `BusinessRuleExpression`, `BusinessRuleSetValueItem`: add
  optional `UId` (`uId`).
- `BusinessRuleAction` hierarchy: add optional `UId` (`uId`) on the base record.

`clio/Command/McpServer/Tools/BusinessRuleTool.cs`
- Mirror the same optional fields on `EntityBusinessRuleMcpContract`,
  `PageBusinessRuleMcpContract`, and the action contracts; thread them through
  `ToBusinessRule()` / `ToBusinessRuleAction()`.

`clio/Command/BusinessRules/BusinessRuleMetadataConverter.cs`
- Honor `rule.Enabled ?? true` and a non-empty `rule.Name` (else generate).
- Honor caller block `uId`s when present (conditions, expressions, actions, set-value
  items); generate otherwise.

`clio/Command/BusinessRules/BusinessRuleAddonService.cs`
- On append: reject a caller-supplied rule name that already exists in the metadata.

## 2. Read path

New `clio/Command/BusinessRules/BusinessRuleMetadataReader.cs` (static, mirror of the
converter): parses the add-on metadata `rules` array into read models —
`BusinessRuleReadResult` records carrying `name`, `caption`, `enabled`,
`convertible`, the friendly `BusinessRule` (when convertible), `esqFilter`
passthrough for apply-static-filter actions, and `raw` metadata JSON otherwise. Skips child
rules (`parentUId`) and derives the parent action's `clearValue`/`populateValue` flags
from them. Reverse maps: comparison int → kebab token,
logicalOperation int → AND/OR, action typeName → kebab action type, `items` CSV → list.
Caption fallback: metadata `caption` → `{uId}.Caption` resource.

`clio/Command/BusinessRules/BusinessRuleAddonService.cs`
- `ReadRules(AddonGetRequestDto)` → fetch schema, return parsed metadata + resources
  (no save).

Services: `IEntityBusinessRuleService.Read(EntityBusinessRulesReadRequest)` and
`IPageBusinessRuleService.Read(...)` — resolve package + schema, build the addon
request (no attribute fetch needed), call reader.

Tools: `ReadEntityBusinessRuleTool` / `ReadPageBusinessRuleTool`
(`clio/Command/McpServer/Tools/BusinessRuleMaintenanceTools.cs`), `ReadOnly = true`. Response:
`{ count, rules: [...] }`.

## 3. Update path

`clio/Command/BusinessRules/BusinessRuleAddonService.cs`
- `UpdateRules(AddonGetRequestDto, IReadOnlyList<BusinessRuleUpdateItem>)`:
  fetch once; per item find rule by name (OrdinalIgnoreCase), replace node in place,
  remove old children, append regenerated children, upsert caption resource; save once.
- uId grafting happens service-side before handing DTOs to the addon service:
  rule/case/group/trigger uIds from the existing metadata node, block uIds from caller
  input (already stamped by the converter), child rules re-anchored to the preserved
  parent uId (`parentUId`, `Autogenerated_{uId}_…` name, `ChildRule-{uId}-…` caption).

Services: `Update(EntityBusinessRulesBatchRequest)` /
`Update(PageBusinessRulesBatchRequest)` reusing validation + conversion from create,
plus a `name is required` per-rule check.

Tools: `UpdateEntityBusinessRuleTool` / `UpdatePageBusinessRuleTool`, destructive.
Response mirrors the create batch response (`updated`/`failed`/`results`).

## 4. Delete path

`clio/Command/BusinessRules/BusinessRuleAddonService.cs`
- `DeleteRules(AddonGetRequestDto, IReadOnlyList<string> ruleNames)`: fetch once,
  remove matched rules + `parentUId` children + caption resources, save once when
  anything matched; return per-name outcomes.

Services: `Delete(EntityBusinessRulesDeleteRequest)` / page counterpart — resolve
package + schema only.

Tools: `DeleteEntityBusinessRuleTool` / `DeletePageBusinessRuleTool`, destructive.
Response: `{ deleted, failed, results }`.

## 5. MCP surface

- `ToolContractGetTool`: six new `ToolContractDefinition` entries + catalog registration.
- `BusinessRulesGuidanceResource`: document the read → update/delete workflow, `name` as
  the match key, uId preservation, and the non-convertible / esqFilter read shapes.
- `BindingsModule.cs`: register the six new tools (and no new services beyond what the
  existing interfaces gain).

## 6. Tests

- Unit (`clio.tests`, `Module=Command` + `Module=McpServer`):
  - `BusinessRuleMetadataReaderTests` — reverse mapping incl. round-trip through the
    converter, child-rule folding, non-convertible fallback, caption resource fallback.
  - `BusinessRuleAddonServiceTests` extensions — update replace/graft, delete cascade,
    caption resource upsert/removal, single save per batch, duplicate-name rejection.
  - Service tests — update matching/not-found isolation, delete outcomes.
  - Tool mapping tests following `BusinessRuleToolTests` patterns.
- E2E (`clio.mcp.e2e`): extend the entity/page business-rule E2E fixtures with
  create → read → update → read → delete flows.
